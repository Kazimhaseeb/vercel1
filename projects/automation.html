PROJECT 01 / SCRAPY · SELENIUM · FLASK

Job Board Aggregator & Tracker

A multi-site job scraping platform that aggregates listings from Rozee.pk, Indeed, and LinkedIn into a single SQLite database, with a Flask dashboard to filter and track applications and email alerts for new matching jobs.

Scrapy Spiders Selenium JS Flask Dashboard Email Alerts SQLite Deduplication

This project tackles a real pain point: job hunting across multiple platforms simultaneously. Scrapy spiders handle static sites like Rozee.pk and Indeed, extracting job titles, companies, links, and dates efficiently. For JavaScript-rendered pages on LinkedIn, Selenium WebDriver takes over, scrolling and waiting for dynamic content to load before parsing. All listings are stored in SQLite with a deduplication layer that checks URL hashes to prevent duplicate entries across runs. A Flask dashboard lets you filter by skill, salary range, or location, and an SMTP-based email alert fires whenever a new listing matches your saved criteria.

How It Works

1
Scrapy Spiders — Static Sites
Scrapy spiders crawl Rozee.pk and Indeed, following pagination and extracting job title, company name, location, salary (if listed), posting date, and job URL from each listing page.
2
Selenium — JS-Rendered LinkedIn
A headless Chrome session via Selenium handles LinkedIn's JavaScript-heavy job feed. The bot scrolls to load more listings, waits for network requests to settle, then extracts the same structured fields via BeautifulSoup on the rendered HTML.
3
Deduplication & SQLite Storage
Before inserting any record, a URL hash is checked against the existing database. Duplicates are silently skipped. New listings are inserted with a default application status of "Not Applied".
4
Flask Dashboard
A lightweight Flask web app renders all stored jobs in a filterable table. Filters for skill keywords, salary range, and location are applied server-side. Application status can be updated directly from the dashboard.
5
Email Alert System
After each scrape run, new listings are matched against saved alert criteria (keywords, location). Matching new jobs are bundled into a formatted HTML email and sent via SMTP so you never miss a relevant posting.

Data Captured

  • Job title & company name
  • Location & salary (where available)
  • Posting date & source platform
  • Job URL & application status

Key Features

  • Multi-site scraping in a single run
  • URL-hash deduplication across runs
  • Dashboard filter by skill / location
  • Automated email alerts on new matches

PROJECT 02 / TESSERACT · OPENCV · PDF2IMAGE

Invoice / Receipt OCR Data Extractor

An end-to-end OCR pipeline that converts scanned PDF invoices into structured data — deskewing and denoising images with OpenCV, extracting text with Tesseract, parsing key fields with Regex, and exporting to CSV or Excel automatically.

pdf2image OpenCV Tesseract OCR Regex Parsing Urdu / Arabic CSV / Excel Export

Manual invoice data entry is error-prone and time-consuming. This pipeline automates the full extraction workflow: pdf2image renders each PDF page as a high-resolution image, OpenCV preprocesses it by deskewing skewed scans, denoising, and applying adaptive thresholding for cleaner OCR input, and Tesseract extracts the raw text. Custom Regex patterns then parse vendor name, invoice date, line-item totals, tax amounts, and grand totals from the raw text output. The extracted data is exported to structured CSV or Excel files automatically. Crucially, Tesseract language packs for Urdu and Arabic are configured to handle bilingual invoices common in Pakistan and Gulf markets.

Pipeline Steps

1
PDF → Image Conversion (pdf2image)
Each page of the scanned PDF invoice is rendered at 300 DPI using pdf2image (Poppler backend), producing a high-resolution PIL Image object per page for downstream processing.
2
OpenCV Preprocessing
The image is converted to grayscale, deskewed using the Hough line transform to correct scan rotation, denoised with a Gaussian blur, and binarised using adaptive thresholding — producing a clean black-and-white image optimised for Tesseract accuracy.
3
Tesseract OCR Extraction
pytesseract runs Tesseract on the preprocessed image with the appropriate language pack (eng, urd, or ara). The raw text output is a string containing all detected characters from the invoice page.
4
Regex Field Parsing
Custom Regex patterns are applied to the raw text to extract: Vendor Name, Invoice Number, Date, Subtotal, Tax percentage and amount, and Grand Total. Patterns handle common formatting variations across different invoice layouts.
5
CSV / Excel Export
Structured field data is written to CSV using the csv module or to a formatted Excel workbook using openpyxl, with column headers, data types, and optional per-vendor sheets.
Multilingual Support

The pipeline is configured with Tesseract language packs for Urdu (urd) and Arabic (ara), enabling accurate OCR on bilingual invoices — critical for Pakistani and Gulf market use cases where vendor names, addresses, and item descriptions mix English with Urdu or Arabic script.

Fields Extracted

Vendor Name — Parsed from invoice header region
Invoice Date — Regex matches DD/MM/YYYY, YYYY-MM-DD, and text dates
Subtotal & Tax — Extracted with numeric Regex patterns
Grand Total — Highest numeric value in the totals region

PROJECT 03 / BEAUTIFULSOUP · APSCHEDULER · TELEGRAM

E-commerce Price Monitor & Alert Bot

An automated price-watching system that scrapes product prices from Daraz and Amazon on an hourly schedule, stores full price history in SQLite, and fires a Telegram notification the moment a price drops below your set threshold.

BeautifulSoup4 requests-html APScheduler Telegram Bot SQLite History matplotlib Charts

This bot lets you track product prices on Daraz and Amazon without ever refreshing the page. BeautifulSoup4 handles standard HTML product pages while requests-html (with its JavaScript rendering capability) tackles JS-heavy product listings. APScheduler runs a price check every hour in the background. Every recorded price is stored in SQLite as a timestamped entry, building a complete price history per product. When the current price dips below your configured threshold, a Telegram notification is sent instantly to your channel. A built-in CLI lets you add or remove tracked products without touching the code, and matplotlib can render a price history chart for any tracked product.

How It Works

1
Add Products via CLI
Run python bot.py add --url "https://daraz.pk/..." --threshold 2500 to register a product. The URL, target threshold, and product name are stored in the SQLite products table. Remove with python bot.py remove --id 3.
2
Scrape Price
For each tracked product URL, requests-html fetches the page (rendering JavaScript for SPAs), then BeautifulSoup parses the price element using CSS selectors or XPath patterns specific to each site.
3
Store to SQLite Price History
The scraped price and current timestamp are inserted into a price_history table, associating the reading with the product ID. This builds a complete timeline of every price recorded.
4
Threshold Check & Telegram Alert
If the scraped price is at or below the product's configured threshold, the Telegram Bot API sends a formatted message to the configured chat/channel with the product name, current price, threshold, and a direct link.
5
Price History Chart
Run python bot.py chart --id 3 to generate a matplotlib line chart of all recorded prices for that product over time, showing trend and price volatility at a glance.

PROJECT 04 / SELENIUM · PYTEST · FAKER

Automated Form Filler & Test Framework

A Selenium-based automated testing framework that generates realistic fake test data with Faker, organises test cases with pytest, captures failure screenshots automatically, and produces a full HTML report after every run — all configurable via a single YAML file.

Selenium WebDriver Headless Chrome pytest Faker HTML Reports YAML Config

Manual form testing across client websites is tedious and error-prone. This framework automates it entirely. Selenium WebDriver launches Chrome in headless mode for speed and runs against any site URL. The Faker library generates realistic names, emails, phone numbers, and addresses for test data — no hardcoded values. pytest organises all test cases cleanly with fixtures, parametrize decorators, and shared conftest setup. On any test failure, a screenshot is saved automatically to a timestamped failure folder. After every run, a full HTML report is generated via pytest-html. A YAML config file stores base URL, form selectors, and test parameters, meaning new client sites can be added without any code changes.

PROJECT 05 / SELENIUM · NLTK · BEAUTIFULSOUP

Social Media Content Scraper & Analyzer

A social media intelligence tool that uses Selenium to scroll and load dynamic post feeds, BeautifulSoup to parse them, and NLTK / TextBlob to score sentiment — with word cloud and bar chart visualisations, date-range tracking, and a scheduled daily email digest.

Selenium Scrolling BeautifulSoup4 NLTK / TextBlob Word Cloud CSV Export Daily Email

Social media feeds are dynamic, JavaScript-rendered, and infinite-scrolling — making traditional HTTP scraping useless. This tool uses Selenium to open a browser, scroll down the page repeatedly to trigger lazy-load events, and wait for new posts to render before extracting. BeautifulSoup then parses the fully-loaded HTML to extract post text, author, timestamp, and engagement metrics. Each post or comment is passed through NLTK's VADER sentiment analyzer (or TextBlob's polarity scorer) to produce a positive / neutral / negative classification and score. Results are visualised as a word cloud of top keywords and a bar chart of sentiment distribution. The full dataset is exported to CSV for further analysis. A scheduled daily scrape emails a summary report automatically.

Analysis Pipeline

1
Selenium Dynamic Scraping
A Selenium session navigates to the target social media page, logs in if required, and executes repeated JavaScript scroll-to-bottom commands with configurable pause intervals to allow new posts to load. A post count check stops scrolling when no new content appears.
2
BeautifulSoup Parsing
The page's rendered HTML source is passed to BeautifulSoup, which extracts post text, author name, post timestamp, and like/comment counts from each post element using CSS selectors or class patterns specific to the target platform.
3
Sentiment Scoring (NLTK / TextBlob)
NLTK's VADER (Valence Aware Dictionary and sEntiment Reasoner) assigns compound, positive, neutral, and negative scores to each post. TextBlob is used as a secondary scorer for polarity and subjectivity. Posts are classified into three buckets: positive, neutral, or negative.
4
Visualisation & Export
wordcloud generates a visual keyword map from all post text (stopwords removed). matplotlib produces a bar chart of sentiment distribution. All data is exported to a CSV file with columns: post_text, author, date, sentiment_score, sentiment_label.
5
Scheduled Daily Digest
APScheduler triggers the full scrape-analyze-report pipeline daily at a configured time. A summary email is generated with total posts scraped, sentiment breakdown percentages, top 10 keywords, and links to the full CSV and charts.

PROJECT 06 / SELENIUM · SQLITE · CSV

LinkedIn Auto-Outreach & CRM Bot

A LinkedIn outreach automation bot that reads target profiles from CSV, personalises connection requests using a name + template system, tracks all outreach in a local SQLite CRM, rate-limits to ~20 actions/day, and emails a daily summary report.

Selenium Human-Like Delays SQLite CRM Rate Limiting Session Cookies Daily Summary Email

LinkedIn's connection outreach is high-value but time-consuming when done manually at scale. This bot automates it safely. Selenium navigates LinkedIn's interface with randomised human-like delays (drawn from a normal distribution) between every action to mimic natural behaviour and avoid detection. Target profiles are read from a CSV file (name, company, role, profile URL), and each connection message is personalised using a name + message template system. A local SQLite CRM tracks the full lifecycle of each outreach: sent, accepted, or replied. The bot hard-caps at ~20 actions per day to stay within LinkedIn's unofficial limits. Browser sessions are saved as cookies, enabling pause-and-resume across days without logging in again. A daily summary email reports total sent, new accepts, and new replies.

Outreach Workflow

1
Load Session & Log In
On first run, Selenium logs in with credentials and saves session cookies to a local file. On subsequent runs, cookies are loaded directly — no login required unless the session expires.
2
Read Target CSV & Filter
The bot reads the target CSV and cross-references against the SQLite CRM to skip any profiles already contacted. Only fresh targets with status "pending" in the CRM are processed.
3
Navigate & Personalise Message
For each target, the bot navigates to their LinkedIn profile URL, clicks "Connect", selects "Add a note", and fills in a personalised message by substituting {first_name}, {company}, and {role} placeholders from the template.
4
Rate Limit & Human-Like Delays
After each send, a random delay between 30–90 seconds is applied. A daily counter is checked before each action; once ~20 actions are reached, the bot saves progress to the CRM and exits, resuming the next day.
5
CRM Update & Daily Email
Each sent request is logged in SQLite with status "sent" and timestamp. The bot also checks recent notifications for accepted connections and replies, updating CRM status accordingly. A formatted summary email is sent at the end of each session.
Safety & Anti-Detection Measures

All delays are drawn from a random normal distribution rather than fixed intervals, making timing patterns indistinguishable from human behaviour. The ~20 actions/day cap aligns with LinkedIn's known informal limits. Browser session cookies prevent repeated login events that can trigger security checks.

PROJECT 07 / SCRAPY · HUGGINGFACE · FASTAPI

Newspaper Aggregator with AI Summarizer

A full-stack news aggregation pipeline that crawls Dawn, Geo, BBC, and Al Jazeera with Scrapy, extracts article content with Newspaper3k, generates concise AI summaries using a HuggingFace transformer, and exposes everything via a FastAPI backend with category filters and a daily digest email.

Scrapy Newspaper3k HuggingFace FastAPI RSS Feed Daily Digest Email

This is the most technically sophisticated project in the collection. Scrapy crawls four major news sources (Dawn, Geo, BBC, Al Jazeera) for fresh articles every few hours. Newspaper3k extracts the full article body, author byline, and publish date from each URL with minimal boilerplate parsing. Each article body is then passed through a HuggingFace transformer model (e.g., facebook/bart-large-cnn or t5-small) to generate a concise, readable summary — turning long articles into 2–3 sentence digests. A FastAPI backend exposes two clean REST endpoints: /headlines (latest articles with metadata) and /summary/{article_id} (the AI-generated summary). A frontend dashboard renders articles with category filters (politics, tech, sports, business). An RSS feed is auto-generated for self-hosting. Finally, a daily digest email delivers the top 5 AI-summarised stories directly to your inbox.

System Architecture

1
Scrapy Crawlers — Multi-Source
Separate Scrapy spiders are written for Dawn, Geo, BBC, and Al Jazeera. Each spider follows pagination and article links, yielding (url, title, category) tuples. A scheduler runs all spiders every few hours. Duplicate URLs are filtered via a seen-URLs set.
2
Newspaper3k Extraction
For each new article URL, Newspaper3k downloads the page, parses out the full article body text, author name, and publish date using its built-in NLP pipeline — far more reliably than manual CSS selectors across different site layouts.
3
HuggingFace AI Summarization
The article body is passed to a loaded HuggingFace summarization pipeline (facebook/bart-large-cnn or t5-small for lighter inference). The model generates a 2–3 sentence abstractive summary. Summaries are stored alongside the article in the database.
4
FastAPI REST Endpoints
GET /headlines?category=tech&limit=20 returns latest articles with title, source, publish date, and URL. GET /summary/{article_id} returns the stored AI-generated summary. Both endpoints support query parameters for filtering and pagination.
5
RSS Feed & Daily Digest
A GET /rss endpoint generates a valid RSS 2.0 XML feed of the latest articles with summaries — compatible with any RSS reader for self-hosting. Each morning, a digest job selects the top 5 most-shared or newest articles, formats them into a styled HTML email with summaries, and sends via SMTP.
AI Summarization Model

The default model is facebook/bart-large-cnn, a BART model fine-tuned on the CNN/DailyMail dataset — purpose-built for news article summarisation. For resource-constrained deployments, t5-small is an alternative that runs efficiently on CPU with slightly shorter summaries. Both are loaded via HuggingFace's pipeline("summarization") API with configurable max/min length parameters.
