Fisico Da Urlo is a gym management ecosystem comprising a Flutter mobile app for members (check-in via QR), a Python FastAPI backend, and this React.js admin panel for gym staff. The admin dashboard is the control centre — administrators manage the full member database and review attendance history through a clean, efficient interface.

Every action in the panel — adding a member, editing a profile, or deleting a record — flows through the same FastAPI endpoints used by the mobile app, ensuring a single source of truth across web and mobile. On member creation, a unique QR code is automatically generated and linked to their profile for gym entry scanning.

Core Features

Secure Admin Login — Password-protected login ensuring only authorised administrators access the dashboard.
Overview Dashboard — Central hub with member list, search, subscription tiers (Bronze/Silver/Gold/Platinum), and status at a glance.
Add New Member — Registration form with personal details and membership type; auto-generates a unique QR code on save.
Edit Member — Pre-populated form to update profiles, membership level, dates, and trigger password reset.
Delete Member — Permanent removal with confirmation step to prevent accidental deletions.
Attendance Tracking — Expandable per-member history showing check-in/out times, reader, location, and present status.

System Architecture

Frontend — React.js

  • Component-based admin UI
  • Protected routes & authentication
  • Member management interfaces
  • Attendance log views & search
  • REST API integration layer

Backend & Data

  • HTTPS REST endpoints
  • Real-time sync with Flutter app

Platform Highlights

Member List & Search

The main dashboard displays all gym members in a sortable table — name, member ID, subscription type, active status, and attendance count. Admins search by name and expand any row for full check-in history.

QR Code Generation

When a new member is created, the system automatically generates and links a unique QR code to their profile — used by the mobile app for gym entry scanning at connected readers.

Ecosystem Integration

The admin panel, Flutter member app, and FastAPI backend share one MySQL database. Changes made in the web panel reflect instantly on mobile — and vice versa — through the unified API layer.

Deployment — Aruba CMP

Both the React static build and FastAPI server run on the same Aruba Cloud Linux VM, minimising latency and simplifying infrastructure. Administrators access the panel via browser with secure HTTPS.

Admin Workflow

1
Administrator Login
Staff authenticate via the secure login screen. Only authorised credentials unlock access to the management dashboard.
2
Review Members & Attendance
The dashboard shows all members with subscription status. Admins search, filter, and expand rows to view detailed attendance history per member.
3
Register New Member
The add-member form captures name, email, phone, membership tier, and dates. Submission saves to MySQL via FastAPI and generates a QR code.
4
Edit or Remove Members
Existing profiles are updated through pre-filled edit forms. Deletions require confirmation before permanent removal from the system.
5
Real-Time Sync
All changes propagate through FastAPI to the shared database — keeping the admin panel and Flutter mobile app consistently in sync.
