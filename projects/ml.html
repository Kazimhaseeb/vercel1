The Healthcare RAG System is a powerful Retrieval-Augmented Generation web application specifically designed for healthcare and medical research documents. It ingests, processes, and queries large datasets — such as the CORD-19 (COVID-19 Open Research Dataset) — using state-of-the-art Large Language Models delivered through a Flask-based Single Page Application.

Users can seamlessly ask questions, generate structured literature reviews, or compare findings across multiple peer-reviewed research papers — all backed by evidence retrieved from a FAISS vector index with numbered source document citations.

Core Features

Intelligent Document Retrieval — FAISS-based vector database enables lightning-fast similarity search across thousands of research documents.
Advanced LLM Integration — Generation powered by Groq (LLaMA-3.1-8b-instant) via LangChain; embeddings by Google GenAI (gemini-embedding-001) for high-quality text representations.
Multiple Query Modes — Q&A mode for direct answers; Literature Review for synthesised topic overviews; Compare mode for contrasting methodologies across sources.
CSV Data Uploads — Upload new CSV datasets through the web UI for ingestion into the vector store without restarting the application.
Modern Web UI — Clean, responsive Vanilla JS/CSS frontend served by Flask with adjustable retrieval settings (Top-K, Temperature).
Rate-Limit Resilience — Built-in retry mechanisms gracefully handle 429 Too Many Requests responses from LLM providers.

RAG Pipeline Architecture

1
Data Ingestion & Cleaning
CSV datasets (CORD-19 metadata) loaded with Pandas, text cleaned and preprocessed. Chunking logic splits documents into optimal segments for embedding.
2
Embedding Generation
Google Gemini (gemini-embedding-001) generates high-quality dense vector representations for each document chunk via LangChain embedding factory.
3
FAISS Vector Indexing
Embeddings stored in FAISS (faiss-cpu) vector database for lightning-fast similarity search. Index persisted to disk for instant application restarts.
4
Query & Retrieval
User query embedded in real time, top-K relevant chunks retrieved by cosine similarity from FAISS index. Configurable Top-K (1–10) and Temperature (0.0–1.0) settings.
5
LLM Generation with Citations
Retrieved context injected into Groq LLaMA-3.1-8b-instant prompt via LangChain. Response includes structured answers with numbered source documents (DOI, date).
System Status Panel

Real-time display of Knowledge Base (faiss_index), document count, LLM provider (Groq), active model (llama-3.1-8b-instant), embedding API (Google Gemini), and embed model (gemini-embedding-001).

Query Modes

Three distinct modes: Q&A for direct factual answers, Literature Review for synthesised topic overviews aggregating multiple sources, and Compare for contrasting research methodologies across papers.

Backend Architecture

  • Python 3.8+ with Flask & Flask-CORS
  • LangChain RAG framework
  • FAISS (faiss-cpu) vector database
  • Groq API — LLaMA-3.1-8b-instant
  • Google Gemini API embeddings
  • Lazy-loaded RAG components

Frontend & Features

  • HTML5, Vanilla CSS, Vanilla JavaScript
  • Configurable Top-K & Temperature
  • CSV drag-and-drop upload (max 200 MB)
  • Expandable source documents panel
  • Session-level query history sidebar
  • Rate-limit retry mechanisms
