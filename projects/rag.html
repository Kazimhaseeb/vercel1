End-to-end RAG systems enabling natural language queries over large document collections. The pipeline ingests PDFs, financial reports, and research papers — chunks and embeds with OpenAI/sentence-transformers, stores in ChromaDB — then retrieves semantically relevant passages to augment GPT-4 completions with source citations. The AI Trading Research Assistant applies this to SEC filings and earnings call transcripts for investment research automation.

RAG Pipeline Architecture

1
Document Ingestion
PDF/DOCX files loaded with PyMuPDF, text extracted and cleaned, metadata preserved (source, page, section, date).
2
Chunking & Embedding
RecursiveCharacterTextSplitter creates 512-token chunks with 50-token overlap. OpenAI text-embedding-3-small generates 1536-dim dense vectors.
3
Vector Storage (ChromaDB)
Embeddings + metadata stored in ChromaDB collections, persisted to disk. Supports hybrid search combining semantic cosine similarity with BM25 keyword matching.
4
Query, Retrieval & Re-ranking
User query embedded, top-K chunks retrieved by cosine similarity, then re-ranked with a cross-encoder for precision before passing to the LLM.
5
Augmented Generation with Citations
Retrieved context injected into GPT-4 prompt via LangChain. Response includes in-text citations with page references. FastAPI endpoint serves the full Q&A pipeline.
AI Trading Research Assistant

Specialized RAG system over 500+ SEC 10-K/10-Q filings and earnings call transcripts. Enables queries like "What did Apple's CEO say about margins in Q3 2023?" with cited, grounded answers.

Hybrid Search

Combines dense vector similarity (ChromaDB) with sparse BM25 keyword matching via rank-bm25. Cross-encoder re-ranking further improves precision before LLM augmentation.
